# Pinpoint support

## Getting help

Before reporting a problem, confirm that you are using:

- VS Code 1.109 or newer
- VS Code's built-in Integrated Browser
- A loaded page in the active Integrated Browser tab

Search the [existing issues](https://github.com/MarcellM01/Pinpoint/issues) first. If the problem has not been reported, [open a new issue](https://github.com/MarcellM01/Pinpoint/issues/new) with:

- Your VS Code version and operating system
- Clear reproduction steps
- What you expected and what happened instead
- Relevant error messages from VS Code's Extension Host output

Do not attach private `.pinpoint/` reports or page content without reviewing and redacting them first.

## Feature requests

Feature ideas are welcome in [GitHub Issues](https://github.com/MarcellM01/Pinpoint/issues). Describe the frontend workflow you want to improve and why the current capture is insufficient.

## Security reports

Do not report vulnerabilities in a public issue. Follow the private reporting instructions in [SECURITY.md](SECURITY.md).
